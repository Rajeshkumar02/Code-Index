# Getting Started with Next.js 15: The Complete Guide

Next.js has revolutionized the way we build React applications, and with version 15, it continues to push the boundaries of what's possible in web development. This comprehensive guide will walk you through everything you need to know to get started with Next.js 15.

## What's New in Next.js 15?

Next.js 15 brings several exciting features and improvements:

### 1. Enhanced App Router

The App Router, introduced as stable in Next.js 13, has received significant improvements in version 15:

- Better error handling and recovery
- Improved streaming and Suspense support
- Enhanced parallel routing capabilities
- More efficient data fetching patterns

### 2. Performance Optimizations

Version 15 focuses heavily on performance:

```javascript
// Improved automatic code splitting
import { lazy } from "react";

const DynamicComponent = lazy(() => import("./DynamicComponent"));

export default function Page() {
  return (
    <div>
      <DynamicComponent />
    </div>
  );
}
```

### 3. Better Developer Experience

The development experience has been enhanced with:

- Faster Hot Module Replacement (HMR)
- Improved error messages and debugging
- Better TypeScript integration
- Enhanced dev tools and diagnostics

## Setting Up Your First Next.js 15 Project

Getting started with Next.js 15 is straightforward. Here's how to create your first project:

### Installation

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

```
├── components/
├── lib/
├── next.config.js
├── package.json
└── tsconfig.json
```

## Key Concepts

### App Router vs Pages Router

Next.js 15 primarily focuses on the App Router, which provides:

- File-based routing with `app/` directory
- Layout composition
- Server and Client Components
- Streaming and Suspense support

### Server Components

Server Components are a game-changer for performance:

```tsx
// This runs on the server
export default async function ServerComponent() {
  const data = await fetch("https://api.example.com/data");
  const result = await data.json();

  return (
    <div>
      <h1>Server-rendered Content</h1>
      <pre>{JSON.stringify(result, null, 2)}</pre>
    </div>
  );
}
```

### Client Components

When you need interactivity, use Client Components:

```tsx
"use client";

import { useState } from "react";

export default function ClientComponent() {
  const [count, setCount] = useState(0);

  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

## Advanced Features

### Middleware

Middleware allows you to run code before a request is completed:

```javascript
// middleware.ts
import { NextResponse } from "next/server";

export function middleware(request) {
  // Add custom logic here
  return NextResponse.next();
}

export const config = {
  matcher: "/dashboard/:path*",
};
```

### API Routes

Create powerful API endpoints:

```typescript
// app/api/users/route.ts
import { NextRequest, NextResponse } from "next/server";

export async function GET(request: NextRequest) {
  const users = await fetchUsers();
  return NextResponse.json(users);
}

export async function POST(request: NextRequest) {
  const body = await request.json();
  const newUser = await createUser(body);
  return NextResponse.json(newUser, { status: 201 });
}
```

## Performance Best Practices

### 1. Image Optimization

Always use the Next.js Image component:

```tsx
import Image from "next/image";

export default function OptimizedImage() {
  return (
    <Image src="/hero-image.jpg" alt="Hero" width={800} height={400} priority />
  );
}
```

### 2. Font Optimization

Use the built-in font optimization:

```tsx
import { Inter } from "next/font/google";

const inter = Inter({ subsets: ["latin"] });

export default function Layout({ children }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

### 3. Bundle Analysis

Analyze your bundle size:

```bash
npm install @next/bundle-analyzer
```

```javascript
// next.config.js
const withBundleAnalyzer = require("@next/bundle-analyzer")({
  enabled: process.env.ANALYZE === "true",
});

module.exports = withBundleAnalyzer({
  // Your Next.js config
});
```
